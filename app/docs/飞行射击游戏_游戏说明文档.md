# 飞行射击游戏 · 游戏说明文档

> 项目《镜像自我 · 人生预演》职业线「宇航员」对应的小游戏
> 玩法代号：`flight`（拟新增）· 拟新增组件：`src/games/AstroBlaster.tsx`
> 对标玩法：飞机大战 / 纵向卷轴射击（STG，参考《雷电》《1942》《Sky Force》）
> 资产策略：**程序化 Three.js 几何体 + WebAudio 合成音效，零外部素材**（与项目其他四款小游戏一致）；视觉复用宇航员线既有星空 / 地球元素

---

## 一、游戏概述

宇航员职业线在叙事中到达「二十五岁」节点后进入小游戏环节。玩家驾驶一艘小型巡航飞船，穿越近地轨道的碎片带与失控无人机群，向远方的空间站方向突围。

> **与现状的关系**：当前宇航员线挂载的是 [Docking.tsx](../src/games/Docking.tsx)（「失重」一维进近对接）。本设计把宇航员线的玩法**重制为纵向飞行射击**，更贴合「飞机大战」的爽快节奏，同时保留「失重 / 深空 / 远方」的叙事气质。两款可二选一，或本作作为主线、Docking 作为隐藏结局玩法。

- **类型**：纵向卷轴飞行射击（Vertical Scrolling Shoot 'em Up）
- **视角**：第三人称俯视，**玩家飞船固定在屏幕下方**，敌人和障碍从屏幕上方涌来
- **主题氛围**：深空、星云、失重、远方的蓝色地球弧线（复用 Docking 的星空 / 地球程序化贴图）
- **核心目标**：在限定时间内尽可能多击毁敌人与障碍、存活到终点，以累计得分结算评级
- **时长**：20 秒（与 race / fps 统一）
- **接入位置**：[GameHost.tsx](../src/components/GameHost.tsx) 在 `career === 'astronaut'` 时挂载（需把 `Docking` 换为 `AstroBlaster`，或新增开关）

## 二、核心玩法

| 机制 | 说明 |
| --- | --- |
| **底部巡航** | 玩家飞船常驻屏幕下方约 1/5 处，可左右移动（建议再开放上下小范围机动），不可飞出屏幕。 |
| **持续射击** | 飞船**自动连射**能量弹向上（无需按键，或按 `空格` 手动射击），火力随拾取升级。 |
| **障碍 / 敌人** | 屏幕上方持续生成：陨石碎片（纯障碍，撞了掉血）、失控无人机（会反击射弹）、精英拦截机（多血、弹幕）。 |
| **拾取道具** | 击毁特定敌人掉落：火力升级（`P`）、护盾（`S`，临时无敌）、回血（`H`）、清屏炸弹（`B`）。 |
| **生命 / 受击** | 初始 **3 格船体（hull）**，每被撞 / 中弹扣 1 格；归零提前结束。受击后短暂无敌闪烁。 |
| **连击倍率** | 连续击杀且不受伤，倍率累加（×1 → ×2 → ×3，封顶），受伤即清零——奖励「无伤走位」。 |
| **二段高潮** | 倒计时过半后进入「碎片风暴」：障碍密度上升、敌机开始齐射，配合心跳音效。 |
| **终点 BOSS** | 最后约 6 秒刷出一艘中型敌舰（有血条），击毁给大量分数；存活到 0s 亦结算。 |

> 设计意图：左手走位、右手（自动）输出，核心乐趣是「在弹幕缝隙里无伤穿行 + 连击滚雪球」——呼应宇航员「在无垠深空里精准、孤独地向前」的叙事。

## 三、操作方式

| 平台 | 操作 |
| --- | --- |
| 键盘 | `← →` / `A D` 横向移动；可选 `↑ ↓` / `W S` 上下机动（限制在屏幕下半区） |
| 射击 | **自动连射**（默认）；或 `空格` 按住手动射击（沿用项目「空格制动」的肌肉记忆位） |
| 炸弹 | `Shift` / `X` 释放清屏炸弹（消耗拾取的 `B`） |
| 鼠标 / 触屏 | 按住屏幕**拖动**飞船跟随手指移动；自动开火；双击 = 放炸弹（移动端兜底，沿用项目的 Pointer 兜底范式） |

- 移动用「目标位置插值」平滑跟随，飞船带轻微俯仰侧倾（`rotation.z`）模拟转向，手感更顺。
- 全程不要求 Pointer Lock（与 fps 不同），避免移动端锁定障碍，开箱即玩。

## 四、场景组件设计（程序化，无外部资产）

> 项目所有小游戏均为「零素材、程序化几何体」，本游戏沿用同一工程约束。下方给出每个组件用 Three.js 基本体的实现映射，方便直接落地。

| 组件 | 程序化实现建议 | 作用 |
| --- | --- | --- |
| **玩家飞船** | `ConeGeometry`（机身，朝上）+ 两个 `BoxGeometry`（机翼）+ `CylinderGeometry`（尾喷）+ 尾焰 `PointLight`（蓝） | 玩家化身，底部巡航 |
| **能量弹** | 共享 `CylinderGeometry`（细长）+ 自发光 `MeshBasicMaterial`（青色），对象池复用 | 持续向上射击 |
| **陨石 / 碎片** | `IcosahedronGeometry` / `DodecahedronGeometry`，顶点扰动做不规则，灰褐色 + 随机自转 | 纯障碍，可被击碎 |
| **无人机** | `CylinderGeometry`（机身）+ 两侧小 `SphereGeometry`，红色自发光 | 会向下射弹的敌人 |
| **拦截机 / 精英** | 更大的 `BoxGeometry` + 装甲板，多血 | 中后期威胁 |
| **BOSS** | 由 `CylinderGeometry` + 多块 `BoxGeometry` 拼装的中型舰体，带血条 | 终点高潮 |
| **敌方射弹** | 红色小球 `SphereGeometry`，自发光，对象池 | 弹幕 |
| **拾取道具** | `OctahedronGeometry` 自转 + 字母贴图（`CanvasTexture` 画 P/S/H/B） | 升级 / 补给 |
| **星空背景** | 复用 Docking 的 1600 点 `Points` 球面星空，缓慢下移制造「向前飞」的视差 | 深空氛围 |
| **地球弧线** | 复用 Docking 的程序化 `SphereGeometry` 地球，置于远处底部做远景 | 远方 / 母星 |
| **爆炸特效** | 命中时一组向外飞散的 `Points` / 小 `Mesh` + `PointLight` 闪一下 + `sfx.hit/crash` | 击毁反馈 |
| **推进尾迹** | 玩家身后持续生成的蓝白粒子拖尾 | 速度感 |

### 4.1 对象池（性能关键）

子弹、敌弹、敌人、陨石、粒子都高频生灭，必须用**对象池**（预分配 + 复用），避免每帧 `new`/`dispose` 抖动。沿用项目现有的「飞过相机后回收到生成端」的滚动复用模式即可。

### 4.2 碰撞

圆形碰撞最省：每个对象存 `(x, y, r)`，玩家弹 vs 敌人、敌人弹 vs 玩家、玩家 vs 障碍，统一用 `(dx²+dy²) < (r₁+r₂)²` 判定。无需物理引擎。

## 五、关卡与系统设计

### 5.1 时间轴与难度曲线

```
0s ─────── 10s ──── 14s ─── 20s
热身期      碎片风暴   BOSS 战   结算
零星陨石+无人机 密度↑、敌机齐射 中型敌舰刷出 RankSplash
火力 1 级   可升到 3 级  击毁+300 分 finishGame
```

- 敌人生成间隔随时间缩短（如 `900ms → 380ms`），与 fps 靶场的难度递增同构。
- 第 10 秒触发一次 `sfx.heartbeat()` + 横幅「碎 片 风 暴 —— 注 意 规 避」。

### 5.2 敌人 / 障碍类型

| 类型 | 血量 | 分数 | 行为 |
| --- | --- | --- | --- |
| 陨石碎片 | 1 | 5 | 直线下落 + 自转，纯障碍，可被打碎 |
| 失控无人机 | 1~2 | 10 | 下移，间歇向下射单发 |
| 拦截机 | 3 | 25 | 较慢，扇形射 3 发 |
| 精英机 | 5 | 50 | 蛇形走位 + 跟踪射弹 |
| BOSS 中型舰 | 40 | 300 | 顶部横移，周期性弹幕，有血条 |

### 5.3 拾取道具

| 道具 | 效果 |
| --- | --- |
| `P` 火力 | 火力等级 +1（1→3 级，单发→三发散射→五发），满级时改为加分 |
| `S` 护盾 | 4 秒无敌 + 飞船外层蓝色光壳 |
| `H` 回血 | 船体 +1（上限 3） |
| `B` 炸弹 | 存入库存，手动释放清屏 + 全场伤害 |

### 5.4 评分与评级

总分 = `Σ(击杀分 × 连击倍率) + 存活秒数 × 2 + BOSS 击毁奖励`。评级在 [titles.ts](../src/lib/titles.ts) 的 `rankOf` 中新增 `flight` 类型（拟）：

| 评级 | 拟定分数阈值 | 称号（沿用宇航员线 `computeTitle`） | 附加条件 |
| --- | --- | --- | --- |
| **S** | ≥ 300 | 仰望者 | 且全程无阵亡（hull 未归零，类比 Docking 的「一次成型」） |
| **A** | ≥ 180 | 失重旅人 | — |
| **B** | < 180 | 观星学徒 | — |

> ⏱ 20 秒时长下击杀总数减少，阈值相应下调（拟定值，需实测微调）。`titles.ts` 的 `flight` 阈值同步设为 `{ S: 300, A: 180 }`。

结算文案示例：`击毁 18 个目标（BOSS 已击沉），最长连击 ×3，剩余船体 2`，3 秒后回调 `finishGame` 进入结局报告。

## 六、技术实现要点

- **渲染 / 光照**：`WebGLRenderer`（antialias + 像素比上限 2）；半球光 + 一束主光；星空 / 地球 / 自发光弹体构成主要亮度来源（深空偏暗）。
- **帧率无关**：所有位移按 `dt`（上限 0.05s）积分，与 race / dock 一致。
- **自动射击节流**：用累计器 `fireAcc += dt; if (fireAcc > fireInterval) { spawnBullet(); fireAcc = 0 }`，`fireInterval` 随火力等级缩短。
- **受击无敌帧**：受击后 `invuln = 1.2s`，期间飞船材质透明度闪烁 + 忽略碰撞。
- **连击倍率**：`multiplier = min(3, 1 + floor(streak / 8))`，受伤 / 阵亡清零 streak。
- **音效映射**（全部复用现有 `sfx`，零新增素材）：
  - 开火 → `sfx.shot()`（节流，避免爆音）
  - 命中敌人 → `sfx.hit()`
  - 击毁 / 撞击 → `sfx.crash()`
  - 拾取道具 → `sfx.nitro()`（火力升级）/ `sfx.click()`（其他）
  - 受伤 → `sfx.wrong()`
  - 碎片风暴 → `sfx.heartbeat()`
  - BOSS 登场 / 通关 → `sfx.ignite()`
  - 结算 → `sfx.confirm()`
- **健壮性**：`new WebGLRenderer` 失败 → `finishGame(0, ...)` 跳过（同其他游戏），保证整条职业线不卡死；提供「跳过」按钮以 B 级结算。
- **资源释放**：卸载时 `cancelAnimationFrame` + 解绑事件 + 遍历 `dispose()` geometry/material + 移除 DOM。

## 七、接入所需的代码改动

若决定让宇航员线采用本玩法，需要改三处（均很小）：

1. **新增组件** `src/games/AstroBlaster.tsx`（仿 [NightDrive.tsx](../src/games/NightDrive.tsx) / [FpsRange.tsx](../src/games/FpsRange.tsx) 的结构：`useEffect` 里建场景 + 主循环 + `finishGame` 回调 + HUD/`RankSplash`）。

2. **[GameHost.tsx](../src/components/GameHost.tsx)**：把第 20 行
   ```tsx
   {career === 'astronaut' && <Docking />}
   ```
   换成（或加开关）
   ```tsx
   {career === 'astronaut' && <AstroBlaster />}
   ```

3. **[titles.ts](../src/lib/titles.ts)** 的 `rankOf` 表新增 `flight` 阈值：
   ```ts
   flight: { S: 300, A: 180 },   // 分数
   ```
   并在组件内 `rankOf('flight', score)`、S 级叠加「零阵亡」判定（参考 Docking 的 `if (rank === 'S' && st.collisions > 0) rank = 'A'`）。

4. **[story.ts](../src/story.ts)** 的 `astronaut.gameHint` 改为飞行射击的操作提示，例如：
   `← → 移动 · 自动射击 · 躲避陨石与弹幕 · 拾取 P 升级火力`

> 称号池（`POOL.astronaut`：仰望者 / 失重旅人 / 观星学徒）和 `introStill / endStill / palette` 都无需改动，直接复用。

## 八、后续可扩展点

1. **弹幕花样**：给拦截机 / BOSS 加预设弹幕轨迹（环形、螺旋、扇形），提升 STG 感。
2. **武器分支**：火力升级到一定等级后，让玩家在「散射 / 激光 / 跟踪导弹」之间二选一，增加 build 深度。
3. **持久高分**：把最高分写入 `localStorage`，作为宇航员线结局报告里的「个人记录」彩蛋。
4. **双人 / 伴飞无人机**：拾取特殊道具召唤一架伴飞僚机协同射击。
5. **与 Docking 联动**：飞行射击作为「突围段」，突围成功后无缝切入 Docking 的「对接段」做收尾，组成两段式宇航员大关。
6. **资产化升级**：若日后引入太空主题 GLB / 精灵包（飞船、陨石、敌机），可直接替换第四节的程序化几何体，与本作架构无缝衔接。
