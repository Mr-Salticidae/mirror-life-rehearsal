# 射击游戏 · 游戏说明文档

> 项目《镜像自我 · 人生预演》职业线「哨兵」对应的小游戏
> 玩法代号：`fps` · 内部组件：[FpsRange.tsx](../src/games/FpsRange.tsx)
> 资产包：`游戏资产/射击游戏/`（`Cyberpunk-character.zip` + `cyberpunk-street.zip`，均为 Craftpix 出品的 2D 精灵图）

---

## 一、游戏概述

哨兵职业线在叙事中到达「二十五岁」节点后进入考核环节。玩家在风雪边境哨所的夜间靶场执行 20 秒**模拟射击考核**——红色标靶是敌人，白色靶是平民，必须在限时内尽可能多击杀敌人，同时避免误伤平民。

- **类型**：第一人称靶场射击（FPS Range / Gallery Shooter）
- **视角**：第一人称，鼠标控制视角（Pointer Lock 锁定鼠标），点击开火
- **主题氛围**：夜间、雪夜、探照灯、冷月光
- **核心目标**：20 秒内取得尽可能高的得分（命中敌人 + 爆头 + 连击加成）
- **接入位置**：由 [GameHost.tsx](../src/components/GameHost.tsx) 在 `career === 'soldier'` 时挂载
- **资产定位**：`射击游戏/` 下的赛博朋克精灵 / 街景资产用于**换肤或衍生关卡**（见第四节），把「雪夜军事靶场」拓展为「赛博朋克街头射击」场景。

## 二、核心玩法

| 机制 | 说明 |
| --- | --- |
| **目标识别** | 红色标靶 = 敌人（击杀得分）；白色靶 = 平民（误伤扣分）。考验「先看清再开枪」。 |
| **爆头加成** | 命中头部得双倍分（`base × 2`），鼓励精准瞄准。 |
| **连击系统（streak）** | 连续命中累加 streak，每发基础分随 streak 递增（`base = 10 + min(streak-1,5) × 2`），最高 +10/发。打偏、误伤、或敌人超时未击中均**清零 streak**。 |
| **平民误伤** | 击中平民：扣 15 分 + 清零 streak + 警告音。 |
| **二段高潮** | 第 10 秒起「人群涌入靶区」：平民生成概率从 28% 跃升到 48%，逼玩家放慢节奏、辨认后再开枪。 |
| **难度递增** | 敌人生成间隔随时间收紧（`1100ms → 600ms`），靶子存活时间有限（2.6~4.8s），越后期越密集。 |
| **计分结算** | 以累计分数结算评级与称号。 |

> 设计意图：这不是「比谁点得快」，而是「比谁在压力下还能分清敌我」——呼应哨兵职业「守夜、克制、一击定生死」的叙事主题。

## 三、操作方式

| 操作 | 键鼠 | 说明 |
| --- | --- | --- |
| 锁定视角 | 点击画面 | 首次点击请求 Pointer Lock；锁定后才开始计时与出靶 |
| 瞄准 | 移动鼠标 | 鼠标移动直接控制视角（yaw/pitch），中心十字准星即弹着点 |
| 射击 | 鼠标左键 | 屏幕中心射线检测命中；枪口闪光（`PointLight` 闪 60ms）+ 射击音效 |

**降级兜底（QA D-02）**：若浏览器不支持 Pointer Lock（部分移动 / 嵌入环境）或用户拒绝锁定，自动切换为**无锁瞄准模式**——鼠标在屏幕的绝对位置直接映射视角，玩法不中断。若完全不想玩，提供「跳过考核」按钮，以 B 级结算，保证整条职业线永不卡死。

| 阶段 | HUD / 提示 |
| --- | --- |
| 未锁定 | 居中大字「守 夜」+ 操作提示 +「跳过考核」按钮 |
| 10s 高潮 | 顶部横幅「人 群 涌 入 靶 区 —— 看 清 再 开 枪」（约 3s） |
| 结算 | `RankSplash` 评级动画 |

## 四、场景组件与资产对应

> ⚠️ **重要架构说明**：当前 [FpsRange.tsx](../src/games/FpsRange.tsx) 是**纯 3D**（Three.js 程序化几何体），而 `射击游戏/` 下的资产是**纯 2D 精灵图**（PNG 透明底）。两者不能直接替换，需选一条整合路径。

### 4.1 现状（3D 程序化场景）

| 场景元素 | 代码实现 |
| --- | --- |
| 地面 / 围墙剪影 | `PlaneGeometry` 地面 + 14 块随机高度 `BoxGeometry` 远景墙 |
| 雪 | 800 个 `Points` 粒子持续下落 |
| 敌人 / 平民靶 | `CapsuleGeometry`（身）+ `SphereGeometry`（头）+ `TorusGeometry`（识别环），红 / 白着色 |
| 光照 | 半球光 + 环境光 + 冷月光（`DirectionalLight`）+ 暖色探照灯（`SpotLight`） |

### 4.2 资产包内容

#### Cyberpunk-character（赛博朋克角色，Craftpix 2D 精灵）
- **总文件 44**：3 个角色（`1 Biker` / `2 Punk` / `3 Cyborg`）+ `PSD` + License
- **每个角色 12 帧动画**（共 36 张 PNG，透明底）：
  `idle`（待机）、`run`（跑）、`jump` / `doublejump`（跳 / 二段跳）、`attack1~3`（攻击三连）、`run_attack`（跑动攻击）、`punch`（拳击）、`climb`（攀爬）、`hurt`（受击）、`death`（死亡）
- **用途**：作为射击游戏的**敌人 / 平民目标**。`idle` / `run` 用于目标出现与移动，`hurt` / `death` 用于命中反馈，可替代当前的「靶子缩小消失」。

#### cyberpunk-street（赛博朋克街景，Craftpix 2D 贴图）
- **总文件 152**：`1 Tiles`(81) + `2 Background`(6) + `3 Objects`(44) + `4 Animated objects`(12) + `PSD`
- **Tiles（81）**：模块化地面 / 墙面 / 平台瓦片，用于拼接射击场地
- **Background（6）**：分层背景图（`Background.png` + `1~5.png`），用于**视差滚动**远景
- **Objects（44）**：街头物件——`Barrel1~4`（油桶）、`Box1~8`（箱子）、`Bench`（长椅）、`Board1~3`（招牌）、`Fence1~3`（栅栏）、`Bucket`（桶）等，可作掩体 / 障碍 / 装饰
- **Animated objects（12）**：动态物件——`Screen1/2`（屏幕）、`Transporter1~3`（传送带）、`Hammer`（锤）、`Card` / `Chest` / `Money`（互动拾取）、`Entry`（入口）、`Platform`（平台），可做交互机关

### 4.3 两条整合路径

**路径 A —— 衍生为独立 2D 射击小游戏（推荐，资产利用率最高）**

用 Canvas 2D 或 DOM 实现「赛博朋克街头射击」，作为哨兵线的隐藏关卡或新职业线：
- **背景层**：`2 Background/*` 做多层视差，玩家不动、背景横向滚动模拟行进
- **地面 / 平台**：`1 Tiles/*` 拼接可站立 / 可掩蔽的地形
- **敌人**：`Cyberpunk-character` 的 Biker / Punk / Cyborg 三种敌人，分别用 `run`（逼近）、`attack`（开火）、`death`（被击毙）帧做逐帧 / 精灵表动画
- **平民**：复用同套角色但去掉武器帧、降低生成概率，作为「误伤扣分」目标
- **掩体 / 障碍**：`3 Objects` 的油桶、箱子、栅栏做可破坏掩体
- **评分**：沿用现有连击 / 爆头 / 误伤公式，仅把 3D 射线命中换成 2D 矩形 / 像素碰撞

**路径 B —— 在 3D 场景中做广告牌（Billboard）精灵**

保留 Three.js 第一人称框架，把 2D 角色作为 `THREE.Sprite` 贴在 3D 空间：
- 用 `SpriteMaterial` + `texture.map` 加载角色 PNG，`Sprite` 始终面向相机
- 通过切换贴图实现 `idle → hurt → death` 帧动画
- 优点：复用现有 Pointer Lock / 射线检测逻辑；缺点：精灵始终正面朝玩家，缺少 3D 体积感

## 五、关卡与系统设计

### 5.1 时间轴与节奏

```
0s ───────── 10s ───────── 20s
正常阶段       高潮阶段       结算
敌 72%/民 28%  敌 52%/民 48%  RankSplash
出靶 1.1s/个   出靶 0.6s/个   finishGame
             （心跳音效触发）
```

### 5.2 目标类型与计分

| 目标 | 命中部位 | 得分 | 备注 |
| --- | --- | --- | --- |
| 敌人（红） | 身体 | `10 + min(streak-1,5)×2` | streak 越高单发越值钱 |
| 敌人（红） | 头部 | 上面公式 × 2 | 爆头 |
| 平民（白） | 任意 | **-15** | 清零 streak |
| 打偏 | — | 0 | 清零 streak |
| 敌人超时未击中 | — | 0 | 清零 streak（惩罚犹豫） |

### 5.3 评分与评级

评级由累计得分决定，阈值定义在 [titles.ts](../src/lib/titles.ts) 的 `rankOf('fps', score)`：

| 评级 | 得分阈值 | 称号（`computeTitle`） |
| --- | --- | --- |
| **S** | ≥ 100 | 风雪守夜人 |
| **A** | ≥ 50 | 边线哨兵 |
| **B** | < 50 | 迷彩新兵 |

> ⏱ 20 秒时长下出靶总数减少（约 15~20 个），阈值相应下调（拟定值，需实测微调）。同步需把 [FpsRange.tsx](../src/games/FpsRange.tsx) 的 `DURATION = 60` 改为 `20`、二段高潮触发点 `elapsed > 32` 改为 `> 10`、横幅判定 `timeLeft <= 28 && > 23` 改为 `<= 10 && > 7`，并把 `titles.ts` 的 `fps` 阈值改为 `{ S: 100, A: 50 }`。

结算文案示例：`命中 12 个标靶（爆头 4 次），最长连击 7，误伤 1 次`，3 秒后回调 `finishGame` 进入结局报告。

## 六、技术实现要点

- **视角控制**：`document.pointerLockElement` + `mousemove` 的 `movementX/Y` 累积 yaw/pitch，pitch 夹紧在 `[-0.6, 0.6]`、yaw 在 `[-1.1, 1.1]`。
- **射击命中**：`THREE.Raycaster.setFromCamera({x:0,y:0}, camera)` 从屏幕中心打射线，`intersectObjects` 取最近命中；通过 `userData.part === 'head'` 判定爆头。
- **计时起点**：**首次锁定视角后才开始计时与出靶**（`startRef`），避免玩家在阅读提示时被白白消耗时间。
- **目标生命周期**：出生时从 0.01 缩放渐入；被击中或超时后边旋转边缩小消失。开场预置 2 个「永生」演示靶（1 红 1 白），让玩家锁定前就看清敌我区别。
- **二段高潮**：用 `surged` 标志保证心跳音效只触发一次；平民概率从 0.28 升到 0.48。
- **健壮性**：WebGL 不可用 → `finishGame(0, ...)`；Pointer Lock 不可用 → 无锁降级；不想玩 → 跳过按钮。三重兜底确保整条线不卡死。
- **资源释放**：卸载时 `exitPointerLock` + 移除全部事件监听 + `dispose()` 几何体 / 材质。

## 八、后续可扩展点

1. **赛博朋克衍生关卡**：按「路径 A」用全套 2D 资产做一条「赛博警察 / 赏金猎人」新职业线，复用本游戏的连击 / 误伤评分公式。
2. **掩体破坏系统**：把 `Objects` 的油桶 / 箱子做成有 HP 的可破坏掩体，命中后切换为破损贴图。
3. **敌人 AI**：用 `run` / `attack` 帧让 Biker / Cyborg 主动逼近或反击，从「静态靶」升级为「遭遇战」。
4. **武器多样性**：利用角色 `attack1~3` 三连击帧，配合不同枪械（手枪 / 霰弹 / 狙击）做差异化手感。
5. **BOSS 战**：用 `Cyborg` 的 `death` / `hurt` 帧设计血条 BOSS，作为射击线结局高潮。
6. **像素级命中判定**：2D 模式下读取精灵 alpha 通道做精确命中（只算不透明像素），避免「打空气也算命中」。
