# 赛车游戏 · 游戏说明文档

> 项目《镜像自我 · 人生预演》职业线「赛车手」对应的小游戏
> 玩法代号：`race` · 内部组件：[NightDrive.tsx](../src/games/NightDrive.tsx)
> 资产包：`游戏资产/赛车游戏/`（`car model.zip` + `car track.zip`，均为 Kenney 出品的低多边形 3D 模型）

---

## 一、游戏概述

赛车手职业线在叙事中到达「二十五岁」节点后进入小游戏环节。玩家扮演一名攒够了改装费、在雨夜环线上试车的年轻车手，需要在 **20 秒倒计时** 内尽可能远地驾驶，同时躲避车流、拾取氮气加速。

- **类型**：第三人称纵向卷轴竞速 / 闪避
- **视角**：车后跟随俯视（相机在车斜后方，看向远方）
- **主题氛围**：雨夜、霓虹路灯、湿滑城市环线
- **核心目标**：在 20 秒内跑出尽可能远的「里程数」（米），以此结算评级
- **接入位置**：由 [GameHost.tsx](../src/components/GameHost.tsx) 在 `career === 'racer'` 时挂载

## 二、核心玩法

| 机制 | 说明 |
| --- | --- |
| **车道闪避** | 路面为 3 车道（`x = -3.2 / 0 / 3.2`）。玩家车在车道间左右切换，躲避迎面 / 同向的车流。 |
| **持续加速** | 基础车速随时间缓慢提升（`base = 26 + elapsed × 0.5`），游戏越往后越快、越难。 |
| **氮气加速** | 路面散布蓝色八面体「氮气球」，吃到后获得 3 秒 ×1.65 倍加速，并扩大相机视场（FOV+8）产生速度感。 |
| **碰撞惩罚** | 撞到对手车：扣减 60 米里程 + 屏幕震动 + 音效；不结束游戏，鼓励继续。 |
| **终点慢镜** | 最后 1 秒进入慢动作（世界速度 ×0.35），相机拉低贴近，强化「冲线」的电影仪式感。 |
| **计分结算** | 以累计里程（米）作为唯一分数，结算评级与称号。 |

> 速度会被平滑插值（`speed → target`），避免氮气瞬间跳变；里程按真实车速积分，因此「少撞车 + 多吃氮气」= 更高里程。

## 三、操作方式

| 平台 | 操作 |
| --- | --- |
| 键盘 | `← →` 或 `A / D` —— 向左 / 向右变道（每次切换一条车道，边界自动夹紧） |
| 鼠标 / 触屏 | 点击 / 触按屏幕**左半**= 向左变道，**右半**= 向右变道（移动端兜底） |

- 变道是「目标车道」式：按下后车辆会平滑横向滑入目标车道，并带轻微侧倾（`rotation.z / y`）模拟转向。
- 不设油门 / 刹车键——车速由系统按时间与氮气自动管理，玩家只专注于**走位**。

## 四、场景组件与资产对应

当前 [NightDrive.tsx](../src/games/NightDrive.tsx) 出于「零外部资产、可即开即玩」的工程约束，所有场景元素用 Three.js 基本体（Box / Cylinder / Sphere / Octahedron / Points）程序化生成。`赛车游戏/` 下的两份资产包正是用来**替换 / 升级**这些占位几何体的官方 3D 模型。

### 4.1 场景元素 → 资产映射

| 场景元素（代码现状） | 对应资产（待接入） | 资产文件 |
| --- | --- | --- |
| 玩家车（红色楔形 Box + 车厢 + 尾灯） | 低多边形跑车 / 赛车模型 | `car model` → `race.glb`、`race-future.glb`、`sedan-sports.glb`、`hatchback-sports.glb` |
| 对手车 / 车流（随机色 Box） | 多种车型，做差异化交通流 | `car model` → `sedan.glb`、`suv.glb`、`taxi.glb`、`van.glb`、`truck.glb`、`police.glb`、`delivery.glb` 等 |
| 路面（Plane + 车道虚线） | 模块化道路段拼接 | `car track` → `roadStraight.glb`、`roadStraightLong.glb`、`roadStraightArrow.glb`（含车道箭头） |
| 路侧灯柱（Cylinder + Sphere 点光源） | 赛场路灯 / 广告牌 | `car track` → `lightPostLarge.glb`、`lightPostModern.glb`、`lightColored.glb`、`billboard.glb`、`billboardDouble.glb` |
| 路缘护栏 / 护墙 | 红白护栏、墙体、栏杆 | `car track` → `barrierRed.glb`、`barrierWhite.glb`、`barrierWall.glb`、`rail.glb`、`fenceStraight.glb` |
| 起点 / 终点门 | 起跑门、旗门、横幅塔 | `car track` → `roadStart.glb`、`roadStartPositions.glb`、`flagCheckers.glb`、`bannerTowerGreen.glb` |
| 观众台 / 赛场建筑（可选氛围） | 看台、帐篷、维修区 | `car track` → `grandStand.glb`、`tent.glb`、`pitsGarage.glb` |
| 氮气球（蓝色 Octahedron） | 暂无对应模型，可保留程序化拾取物或用 `debris-*` / 锥桶替代 | `car model` → `cone.glb`；`car track` → `pylon.glb` |

> 三个独立「主题地皮」可选：`*Border`（路缘带）、`*Sand`（沙地）、`*Wall`（护墙），可按场景风格切换。

### 4.2 接入方式建议

- **格式选择**：浏览器侧统一用 **GLB**（自包含贴图、单文件、`GLTFLoader` 一步加载）。资产包同时提供了 FBX / OBJ / DAE / STL，仅作离线编辑 / 引擎迁移备用。
- **加载器**：`three/examples/jsm/loaders/GLTFLoader.js`，对赛道模块可在初始化时批量 `loadAsync` 后 `clone()` 复用，避免重复解析。
- **滚动复用**：现有代码用「超过相机后回收到远处重排」的对象池模式（`dashes` / `poles` / `traffic` / `nitros`）。换成模型后沿用同一套对象池即可，把生成函数 `mkTraffic / mkNitro` 内的 `BoxGeometry` 换成 `gltf.scene.clone()`。
- **碰撞**：维持当前的「同车道 + Z 轴距离阈值」AABB 风格判定（`|dz| < 2.6 && |dx| < 1.5`），不必依赖物理引擎。

## 五、关卡与系统设计

### 5.1 难度曲线

```
时间轴  0s ─────────── 20s
车速    26 m/s（≈94 km/h）──缓升──▶ 越来越快
氮气    随机散布（3 个同时存在，吃掉即重排到远端）
车流    7 辆常驻 + 随机补充（速度 16~24，比玩家慢，形成「超车」）
慢镜    最后 1s 触发
```

### 5.2 障碍 / 互动对象

| 对象 | 行为 | 反馈 |
| --- | --- | --- |
| 对手车（traffic） | 在 3 车道随机生成，比玩家慢，玩家不断超越 | 撞击 → 扣 60m + 震动 + `sfx.crash()` |
| 氮气球（nitro） | 随机车道散布，旋转自转 | 拾取 → 3s ×1.65 加速 + FOV 扩大 + `sfx.nitro()` |

### 5.3 评分与评级

评级由累计里程决定，阈值定义在 [titles.ts](../src/lib/titles.ts) 的 `rankOf('race', meters)`：

| 评级 | 里程阈值 | 称号（`computeTitle`） |
| --- | --- | --- |
| **S** | ≥ 600 m | 夜环线之王 |
| **A** | ≥ 450 m | 雨夜车手 |
| **B** | < 450 m | 弯道新人 |

> ⏱ 20 秒时长下里程上限相应降低（无伤氮气流约 700~850m），上述阈值已随之重新标定（拟定值，需实测微调）。同步需把 [NightDrive.tsx](../src/games/NightDrive.tsx) 的 `DURATION = 60` 改为 `20`、`titles.ts` 的 `race` 阈值改为 `{ S: 600, A: 450 }`。

结算文案示例：`冲出 620 米，氮气 3 次，碰撞 1 次`，3 秒后回调 `finishGame` 进入结局报告。

## 六、技术实现要点

- **渲染**：`THREE.WebGLRenderer`（antialias + 像素比上限 2），雾效（`Fog 0x04060d, 30~160`）做远景消隐。
- **光照**：半球光 + 环境光打底；玩家车带一盏 `SpotLight` 车头灯照向远方；路灯光柱用 `PointLight`。
- **氛围层**：900 个雨粒子（`Points`）持续下落并随车速前移回收，营造雨夜与速度。
- **帧率无关**：所有位移按 `dt`（`realDt`，上限 0.05s）积分，慢镜时 `dt *= 0.35`。
- **健壮性**：`new WebGLRenderer` 失败时直接 `finishGame(0, ...)` 跳过，保证整条职业线不因设备不支持 WebGL 而卡死。
- **资源释放**：组件卸载时 `cancelAnimationFrame` + 遍历 `dispose()` geometry / material + 移除 DOM，避免内存泄漏。

## 八、后续可扩展点

1. **完整闭合赛道**：用 `roadCurved` + `roadCornerLarge*` 拼出环形赛道，从「无限直道闪避」升级为「圈速计时」。
2. **多车型车流**：把对手车随机替换为 `taxi / truck / police` 等，让交通流视觉更丰富。
3. **氮气槽可视化**：把瞬时氮气（`state.nitro`）做成 HUD 能量条，并加入「连吃叠加时长」机制。
4. **天气 / 昼夜切换**：资产含 `*Sand` 沙地变体，可做「白天沙漠赛道」作为赛车手结局后的隐藏关卡。
5. **碰撞物理感**：撞车后给对手车一个翻滚动量（沿用 `debris-*` 碎片模型飞溅），强化反馈。
