## 2. Prompt 结构说明（不用复制，理解即可）

每条 prompt 按 Seedance 图生视频的最佳实践构造：**只描述"要动什么"，不复述画面内容**
（场景信息首帧图已提供，复述会诱发重绘漂移）。结构 = 镜头锁定 → 主体微动 → 氛围运动 → 硬约束。
每条末尾的硬约束一致：单镜头、禁转场、禁新增、面部稳定、匀速可循环。

## 4. 审片标准（每条生成后自查，不合格就重roll）

- [ ] 没有自动运镜/转场/切镜（Seedance 2.0 爱自己加戏，出现即废片重roll）
- [ ] 人脸/手没有变形、融化、鬼影（**最常见废片原因，重点看**）
- [ ] 没有凭空长出新物体、新人物、字幕或水印
- [ ] 首尾画面接近（循环播放时跳变不刺眼；轻微跳变可接受，工具有"循环"选项就勾上）
- [ ] 动作幅度小而自然（宁可"几乎不动"也不要"动作夸张"）
- [ ] 构图与原图一致（游戏里视频直接叠在原图上，构图跑了会穿帮）

## 5. 体积控制（重要：13 张全上时总量控制在 ~60MB 内）

单条 ≤5MB 为佳。工具导出过大时用 ffmpeg 压一遍：

```
ffmpeg -i 输入.mp4 -an -c:v libx264 -crf 26 -preset slow -vf scale=1280:-2 输出.mp4
```

⚠️ **ffmpeg 中文路径易报错**：先把文件放到纯英文临时目录（如 `C:\temp\`）处理，再拷回项目。

## 6. 交付流程

1. 改名为表格中的**输出文件名**（小写、下划线，精确匹配）
2. 放入 `app/public/videos/`
3. 本地验证（可选但推荐）：`app` 目录跑 `npm run dev` → 走到对应场景看视频是否播放、有无穿帮
4. 提交：`git pull` → `git add app/public/videos` → `git commit -m "动态剧照: xxx"` → `git push`（被拒先 `git fetch` + `git rebase`）
5. 更新内测站：仓库根目录跑 `.\deploy-preview.ps1`（你们已有预览仓权限，记得先接受 GitHub 邀请）
