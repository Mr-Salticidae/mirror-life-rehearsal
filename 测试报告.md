## 一、测试概述

- **目标**：以测试工程师视角，对预览站的全部功能做端到端验证，确认核心链路可用、定位缺陷、给出发布建议。
- **范围**：待机吸引 → 序幕 → 叙事节点 → 章间分支图 → 职业前奏 → 迷你游戏 → 终幕报告 → 重新预演，以及路由、静音、离线兜底、控制台稳定性。
- **方法**：浏览器自动化驱动；用 `?career=soldier|painter|racer|musician|astronaut` 高效触达各结局线；对叙事段做一次 A→B→F 短流程实测（含一次 QTE 超时观察）；每个游戏只验"加载 + 机制响应 + 能进报告"，不刷分。

## 二、测试内容（覆盖项）

1. **待机吸引模式 attract**：标题、CTA「触碰开始预演」、顶栏声音开关、今日人生墙。
2. **序幕 prologue**：按住擦拭镜面点亮。
3. **叙事系统**：打字机逐字字幕、点击加速、选择支交互、选后后果台词、回响台词（callback）。
4. **QTE 限时 / 长按**：B(8s) / G(8s) / D(10s) 倒计时与超时犹豫；D1 长按 1.2s 推门。
5. **章间分支图**：interlude 自动推进、final 模式全节点矩阵。
6. **5 条结局线**：哨兵(FPS 射击) / 画家(喷涂) / 赛车手(驾驶) / 音乐人(节奏打击) / 宇航员(轨道对接)。
7. **终幕报告**：称号、三段叙事、finalWord、人格四轴、保存分享卡(下载)、重新预演、OFFLINE MODE 标签。
8. **路由与边界**：`?career=` 5 个参数、声音开关、120s 闲置回待机、控制台错误。

## 四、缺陷清单（按严重度排序）

> 严重度：**High**=影响核心体验/可能阻断；**Medium**=明显瑕疵；**Low**=细节优化。

### 🔴 D-01 〔High · 资产级〕核心剧照 13+ 张全部 404，预览站退化为"纯文字 + 无声"
- **描述**：叙事节点剧照、5 条职业前奏剧照、画家游戏背景墙、4 张结局剧照均未部署，控制台持续 404。
- **涉及**：`stills/node_a/b/c/f.jpg`、`stills/career_soldier/painter/racer/musician/astronaut.jpg`、`stills/wall.jpg`、`stills/end_painter/racer/musician/astronaut.jpg`。
- **复现**：进入任意叙事节点 / 职业前奏 / 画家游戏 / 结局报告，控制台报 404；画面以纯色或渐变兜底。
- **预期**：作为"互动影游"，剧照为核心视觉资产，应随节点/职业/结局呈现。
- **实际**：占位兜底**正常工作**（流程不断，符合 [DESIGN.md](DESIGN.md) 的缺图回退设计），但预览观感为纯文字叙事。
- **定性**：**资产就绪度问题，非代码缺陷**。代码兜底符合设计；问题是 MJ 剧照尚未产出/提交（清单见 [assets/PROMPTS.md](assets/PROMPTS.md)）。
- **建议**：按 PROMPTS.md 出图后放入 `app/public/stills/`，重新部署即生效。

### 🔴 D-02 〔High · 行为缺陷〕哨兵线 FPS 在 pointer lock 不可用时卡死、无兜底
- **描述**：`?career=soldier` 进入游戏后，pointer lock 报错 `The root document of this element is not valid for pointer lock`，60s 计时器始终不动，游戏永不结束，**无跳过/退出按钮**。
- **复现**：`?career=soldier` → 触碰进入 → 观察控制台与计时器。
- **预期**：pointer lock 失败时应有超时或退出兜底，避免用户卡死。
- **实际**：整条哨兵线卡死，5 结局中 1 条无法走通。
- **影响**：展位浏览器若拿不到 pointer lock（iframe 预览/部分浏览器/特定策略），哨兵线直接废线。
- **建议**：document/body 级 pointer lock 校验 + 加 60s 硬超时或"跳过进报告"兜底。改动文件：`app/src/games/FpsRange.tsx`。

### 🟠 D-03 〔Medium · 资产级〕背景音乐 bgm.mp3 404，声音开关形同虚设
- **描述**：每次加载控制台报 `audio/bgm.mp3 404`。
- **预期**：顶栏"声音 开/关"控制 BGM 播放。
- **实际**：音频源不存在，开关可切但无声场；BW 展位氛围感丢失。音效为 WebAudio 程序合成，不受影响。
- **建议**：产出 BGM 放入 `app/public/audio/bgm.mp3`（可无，DESIGN 已声明静默兜底）。

### 🟡 D-04 〔Low · 设计澄清〕待机页今日人生墙在无历史时不渲染
- **描述**：首页快照仅含「标题 / CTA / POWERED BY」，无今日人生墙。
- **定性**：**符合代码设计**——人生墙仅在本机 localStorage 有历史时渲染（`hall.length > 0` guard）；Playwright 全新会话无历史，故正确不显示。**非缺陷**。
- **建议（可选）**：展位首启无历史时墙也为空，可加一个空状态提示（如"成为第一个上墙的人"），提升首启观感。

### 🟢 D-05 〔Low〕分享卡 overlay 遮挡「重新预演」按钮
- **描述**：报告页点"保存分享卡"→ share-overlay 弹出后会拦截「重新预演」的点击（`intercepts pointer events`），需先点空白处关闭。
- **建议**：overlay 自动关闭，或将操作按钮置于 overlay 之上。

### 🟢 D-06 〔Low〕直跳模式报告的节点矩阵显示「???」
- **描述**：`?career=` 直跳通关后，报告页 8 节点 × 选项矩阵全显示「? / ？？？」。
- **定性**：直跳跳过叙事，节点无数据属预期。
- **建议**：直跳模式下隐藏该区块或显示"调试模式·无叙事数据"，避免观众误解为 bug。

### 🟢 D-07 〔Low〕favicon.ico 404
- **描述**：浏览器标签无图标，每次导航重复报错。
- **建议**：补一个 favicon。

## 七、结论与发布建议

**核心链路可用**：待机→擦镜序幕→叙事(打字机/选项/回响/章间过渡)→职业前奏→迷你游戏→报告(称号/三段叙事/finalWord/四轴/分享卡/重新预演)→静音 全链路通畅；**4 条结局线（画家/赛车手/音乐人/宇航员）完全可玩可报告**；路由 / 离线模板兜底 / OFFLINE MODE 均符合设计；无未捕获异常。

**阻断性问题**：
1. **哨兵线 FPS（D-02）**：pointer lock 不可用时整线卡死、无退出——必须在真实展位浏览器复测并加超时/退出兜底。
2. **视觉资产（D-01/D-03）**：13+ 张剧照与 BGM 未部署。代码兜底正常，但作为"互动影游"预览，目前是纯文字 + 无声——展区观感大打折扣。

**发布判定：有条件可发布**
- ✅ 若 BW 现场仅开放 4 条非 FPS 结局线，或现场浏览器确认 pointer lock 可用 → 现在可发布（接受预览期无剧照/无声）。
- ⏳ 若展位要求 5 线全开 + 视觉沉浸 → **建议先修 D-02（FPS 兜底）+ 部署 D-01（剧照）/D-03（BGM）再上线**。
- 后续优化：D-04 空状态提示、D-05 overlay 自关闭、D-06 直跳占位、D-07 favicon。
