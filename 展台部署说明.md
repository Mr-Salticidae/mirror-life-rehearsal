# 展台部署说明（BW 展会 · 网盘压缩包版）

> 目标：在展会机器上零环境、零联网跑起《镜像自我·人生预演》完整版。
> 压缩包内已带一切：源码、构建产物、依赖、便携 Node 运行时、剧本文档、git 历史。

## 一、机器要求

- Windows 10 / 11 x64（项目按 RTX 独显机调优，核显也能跑，WebGL 失败会自动跳报告兜底）
- 装有 Chrome 或 Edge（Windows 自带 Edge 即可）
- **无需安装 Node、无需 npm install、无需联网**

## 二、部署三步

1. 从网盘下载压缩包，**解压到本地任意目录**（路径可含中文/空格；不要在压缩软件里直接双击运行）。
2. 进入解压后的项目文件夹，双击 **`双击启动展台.bat`**。
   脚本会自动：用包内 `tools\node.exe` 起本地服务（端口 5174）→ 用 Chrome/Edge kiosk 模式全屏打开。
3. 看到待机页即部署完成。日常重启展台=重复第 2 步（重复双击会自动复用已开的服务）。

**退出**：`Alt+F4` 关全屏浏览器 → 关掉黑色服务窗口（及最小化的 node 窗口）。

## 三、可选：接 LM Studio 真 AI 报告

不接也**完整可玩**——报告自动走离线模板+模拟流式（观感一致，这就是断网态设计）。要接真流式：

1. 展台机装 LM Studio，装载一个中文模型，启动本地服务（默认 `http://127.0.0.1:1234/v1`）。
2. 如端口/模型名不同，改 **`app\dist\config.json`**（改这个立即生效，无需重建；`app\public\config.json` 是源，重建时会覆盖 dist）：
   ```json
   { "baseUrl": "http://127.0.0.1:1234/v1", "model": "改成LM Studio里的模型名", "timeoutMs": 20000 }
   ```
3. AI 断线/超时自动回落模板，不会卡死现场。

## 四、二维码 / 手机报告页

- 观众扫海报或侧栏二维码 → **用观众手机自己的流量**打开
  `https://mr-salticidae.github.io/mirror-life-rehearsal-preview/r.html#...`，展台机全程无需联网。
- ⚠️ BW 结束前不要删预览小仓库，否则观众已保存海报上的码全部失效（详见 PROGRESS.md 待办 7）。
- 如要换报告页域名：改 `app\dist\config.json` 加 `"reportBase": "https://新域名/"`。

## 五、现场排障速查

| 现象 | 处理 |
|---|---|
| 双击 bat 报"服务未就绪/端口占用" | 5174 被其他程序占用：关掉占用程序，或任务管理器结束残留 node.exe 后重试 |
| 没装 Chrome/Edge | 任意浏览器手动开 `http://127.0.0.1:5174/`，按 F11 全屏 |
| 没声音 | kiosk 启动参数已放开自动播放；若仍无声，点一下屏幕即恢复（浏览器手势策略） |
| FPS 游戏鼠标不锁定 | 有无锁降级兜底，可正常玩；键盘游戏全部有鼠标/触屏兜底 |
| 画面全黑/WebGL 报错 | 自动跳报告页兜底；检查显卡驱动，浏览器开硬件加速 |
| 闲置回待机 | 设计使然：120 秒无操作自动回待机页 |
| 直达某职业线调试 | 地址栏加 `/?career=soldier\|painter\|racer\|musician\|astronaut` |

## 六、改代码 / 重建（展会现场一般用不到）

包内 `node_modules` 齐全。便携 `tools\node.exe` 只带运行时不带 npm，要跑 `npm run build` 请安装正式 Node 20+，然后：

```powershell
cd app
npm run build   # tsc -b && vite build → 产物进 app/dist
```

其他日常入口：开发 `npm run dev`（5174）；内测预览站重部署 `.\deploy-preview.ps1`（需要主仓库推送权限与外网）。

| `.git\` | 完整提交历史（可直接继续开发） |
| `node_modules\` | 依赖齐全（离线可重建，见第六节） |
