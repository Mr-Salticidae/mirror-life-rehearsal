# 《镜像自我·人生预演》展台一键部署包

> 部署包 v1.1（2026-07-09，健壮性升级版；应用本体仍为封版 v1.0）。
> 本包**自带全部运行时**，目标机器无需安装 Node/npm/任何依赖。

## 目录结构

```
镜像自我_展台部署包/
├─ 启动展台.bat        ← 双击这个就开演（起服务 + kiosk 全屏浏览器）
├─ 停止展台.bat        ← 收摊时双击（连 kiosk 浏览器一起关）
├─ 展台自检.bat        ← 装机后/开摊前双击，逐项体检
├─ 环境加固.bat        ← 装机时以管理员运行一次（电源/屏保/暂停更新）
├─ 现场SOP.md          ← 打印给现场非技术同事的一页纸
├─ server.js           ← 本地静态服务器（零依赖）
├─ app/                ← 应用本体（构建产物 + 全部素材，约 90MB）
│   └─ config.json     ← ★ 模型接口配置（见下文）
├─ standby/            ← ★ 止血预案（预置构建，现场免编译）
│   ├─ 切换_下线哨兵线.bat  ← pointer lock 真机失败/触屏机时双击
│   └─ 切换_恢复五线.bat
└─ tools/node/node.exe ← 便携版 Node 运行时（仅此一个文件）
```

## 一、开机部署（3 步）

1. 把整个文件夹拷到目标机器任意位置（**路径不要带中文**最稳，如 `D:\MLR\`）
2. 双击 `启动展台.bat` —— 起服务（就绪探测通过后）再全屏打开；
   浏览器优先 Chrome（含用户级安装），无则用系统自带 Edge
3. 完事。闲置 120 秒自动回待机循环，无人值守可跑一整天

- 退出全屏：`Alt+F4`；停止服务：双击 `停止展台.bat`
- 换端口：编辑 `启动展台.bat` 里的 `set PORT=8420`
- 排障模式：命令行 `启动展台.bat nobrowser` 只起服务不开浏览器
- 服务进程带守护：node 崩溃/被误杀 2 秒内自动拉起；双击两次启动器不会撞端口

## 一点五、★ 止血预案（哨兵线下线，30 秒生效）

真机验证 pointer lock 失败、或展台是触摸屏一体机时：

1. 双击 `standby\切换_下线哨兵线.bat`
2. 展台屏幕按 F5 —— 立即变为四线版（画家/赛车手/音乐人/宇航员）

无需构建环境、无需重启服务，`切换_恢复五线.bat` 随时切回。
原理：包内预置了五线/四线两套构建产物，切换只替换 html 入口文件。

## 二、★ 模型替换接口（对接更强 LLM 看这里）

AI 报告走 **OpenAI 兼容协议**（`POST {baseUrl}/chat/completions`，SSE 流式）。
只要端点兼容该协议即可即插即用——改 `app/config.json`、展台页面刷新一次即生效（**无需重启服务，无需重新构建**）：

```json
{
  "baseUrl": "http://127.0.0.1:1234/v1",
  "model": "local-model",
  "apiKey": "",
  "timeoutMs": 20000,
  "temperature": 0.9,
  "maxTokens": 700,
  "reportBase": "https://tiaozhuxiansheng.com/mlr/"
}
```

| 字段 | 说明 |
|---|---|
| `baseUrl` | 端点基址。本地：LM Studio 默认 `http://127.0.0.1:1234/v1`、Ollama `http://127.0.0.1:11434/v1`、vLLM `http://127.0.0.1:8000/v1`；云端中转直接填 https 地址 |
| `model` | 模型名，照端点方要求填（如 `qwen2.5-14b-instruct`、`deepseek-chat`） |
| `apiKey` | 云端密钥。本地端点留空。**⚠️ 只写在展台机的这个文件里，绝不提交进 git / 不进公开预览站** |
| `timeoutMs` | 首包超时（毫秒）。云端网络差可放宽到 30000；流式开始后另有 15s 块间停滞保护 |
| `temperature` / `maxTokens` | 采样温度 / 生成上限。默认 0.9 / 700 按本地 14B 调校；接更强模型可自行调整（报告约需 400 中文字，maxTokens 建议 ≥600） |
| `reportBase` | 扫码报告页基址（观众手机打开的站点）。默认自定义域名 /mlr/（大陆移动网络可达）；github.io 预览站为备用镜像，换域名时改这里 |

**兜底机制**：端点不可用/超时/返回格式异常 → 自动切离线模板报告（报告卡右上角显示
OFFLINE MODE），全流程不断，观众无感知。所以**AI 端点挂了不影响展出**。

**上新模型的验收动作**：跑一局到报告页，确认 ① 右上角显示 RTX LOCAL AI（非 OFFLINE MODE）
② 三段叙事+结语完整 ③ 遥测行"字/秒"在跳。控制台（F12）搜 `[ai]` 可看失败原因。

## 三、目标机器要求

- Windows 10/11 x64，独显（RTX 系）为佳——三个 3D 小游戏需要 WebGL；GPU 弱时游戏自动降级不崩
- 浏览器：Chrome（推荐）或系统自带 Edge；1920×1080 横屏
- 本地 AI（可选）：LM Studio / Ollama 需另行安装并加载模型，**不在本包内**；不装也能跑（离线模板态）
- 打印（可选）：A4 打印机装好驱动设为默认即可，卡片/完整报告两种版式在报告页按钮触发

## 四、故障速查

| 症状 | 处置 |
|---|---|
| 双击启动无画面 | 看任务栏有无 MLR-Server 窗口；没有则路径含中文/被杀毒拦截，换纯英文路径重试 |
| 有画面无声音 | kiosk 参数已带自动播放豁免；若仍无声，点一下屏幕即解锁（浏览器策略） |
| 报告一直 OFFLINE MODE | 本地 AI 未启动或 config.json 的 baseUrl/model 不对；F12 控制台搜 `[ai]` |
| 视频不播只有静图 | 属正常兜底（该场景视频缺失/损坏时自动用剧照）；检查 app/videos/ 对应文件 |
| 游戏卡死 | 不用管：单局 180s / 无操作 60s 看门狗会自动回待机 |
| 手机扫码打不开 | 观众手机需有网（报告页在线上）；展台本身断网不影响扫码内容生成 |
