## 二、内容基准（写任何东西前先读这节）

现有故事骨架（v2 + 迭代一，代码里的最终事实以 `app/src/story.ts` 为准）：

- **主角**：无名无脸（全程背影/剪影），性别中立，玩家即主角。
- **结构**：三章八节点——七岁（巷口/作文课/储蓄罐）→ 十七岁（天台/走廊/深夜门缝）→
  二十二~二十五岁（出租屋/镜前）。每节点 2~3 个选择。
- **数值**：三条暗倾向 `勇`(军人)/`彩`(画家)/`驰`(赛车手)，选择加值，最高者定职业；
  另有 `遗憾` flag、`犹豫`(超时)计数 → 犹豫≥2 或三值全平触发隐藏结局**无名者**。
- **回响机制**：后面的节点会"记得"前面的选择，多出一句台词（每条回响只挂**单一**前置条件）。
- **结局**：职业迷你游戏（FPS/涂鸦/赛车）→ S/A/B 评级 → 分档称号 → AI 人生报告。

### 文风铁律（违反即打回）

1. 第二人称"你"，克制、电影感，**不用感叹号**，不说教，不心灵鸡汤。
2. 台词每句 ≤ 30 字，一个节点基础台词 ≤ 3 句（展会节奏，单轮红线 6 分钟）。
3. 具体物件优先于抽象情绪：写"铁盒打开的那一声"，不写"你感到无比激动"。
4. 选择的三个选项之间必须**等价有吸引力**——不设"正确答案"。
5. 后果文本一句话，落在"多年以后仍会记得的细节"上。

## 三、三个工作包（自行认领，主导拍板）

### 工作包 A：故事扩展（现有三线加深）

- 为现有 8 个节点补**回响台词**（条件用 choiceId，如"若 A2 → C 节点多一句……"）。
- 为三条职业线各写 2~3 个**结局文本变体**（按 S/A/B 评级、遗憾、换门组合）。
- 无名者线扩写：不同触发方式（连续犹豫 vs 三值全平）的报告差异。
- 新节点提案（每章最多再 +1，必须是 10 秒内能完成的"轻节点"）。

### 工作包 B：角色扩展（新职业线构思）

每条新职业线 = 完整提案，包含：
职业名 / 三个年龄段的种子意象（七岁看见什么、十七岁藏着什么、二十二岁买什么）/
对应倾向值命名（一个字）/ 结局迷你游戏概念 / 称号三档 / 报告模板三段+金句。
**候选方向参考**：音乐人、厨师、宇航员、医生、电竞选手——不限于此。
⚠️ 新职业=新迷你游戏，实现成本高，提案会按「实现难度分级」排队（见工作包 C 的分级表）。

### 工作包 C：游戏策划（迷你游戏设计案）

- 现有三游戏的**深化案**：新机制、新演出、平衡性（当前评级阈值：FPS S≥140 / 涂鸦覆盖率 S≥10% / 赛车 S≥1650m）。
- 新迷你游戏案（配合工作包 B），每案必须标注实现难度自评：
  - **L1**（2D canvas / 纯 DOM，如涂鸦）：最容易被采纳
  - **L2**（简单 3D，固定视角，如赛车）
  - **L3**（3D + 视角控制，如 FPS）：展前只在必要时做
- 硬约束：单局 60~90 秒、键鼠操作、10 秒内看懂规则、失败也有正反馈。

## 四、交付格式

`content/` 目录下按模板新建 Markdown 文件，命名：`{类型}_{简名}_{作者缩写}.md`
例：`节点_旧照片_zy.md`、`职业线_音乐人_db.md`、`游戏案_节奏按键_db.md`
模板见 [content/TEMPLATES.md](content/TEMPLATES.md)。

## 安全红线（所有人，包括代码侧）

1. **`app/public/config.json` 永远不放 `apiKey` 或任何密钥。** 展台 AI 走本地端点（127.0.0.1）不需要 key；
   预览站是**公开仓库**，`deploy-preview.ps1` 会把 config.json 原样发布上网。
   部署脚本已加保险（发布前自动剔除 apiKey 字段），但第一道防线是根本不写进去。
2. 任何 key / token / 密码都不进 git（主仓库虽私有，同样禁止——防口径混乱与历史残留）。
3. 预览站上的一切（含剧本文本，它编译在 JS 里）都视为公开内容；不希望公开的东西不要部署上去。

## 五、协作流程

1. `git clone https://github.com/Mr-Salticidae/mirror-life-rehearsal.git`
2. 从 `main` 拉分支：`content/你的名字/简述`（例 `content/zy/musician-line`）
3. 只增改 `content/` 下的文件，提交后发 **PR 到 main**
4. 主导审核：文风铁律 + 硬约束 → 通过后代码侧负责集成进引擎
5. **不要直接推 main**；`app/` 目录的一切修改由代码侧完成
6. 想看效果：找主导要预览链接或录屏；采纳的内容会在集成 commit 里 @ 你
