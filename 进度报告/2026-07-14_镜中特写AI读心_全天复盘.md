## 方法论沉淀

### 1. 纯文本行协议 + 段名固定在客户端

**核心**：让 AI 只产可变内容（副标题/要点），版式骨架（段名/序号/结构）由客户端硬编码——坏输出砸不了版面。
**来源**：closeup 五段协议 `NN|副标题` + `- 要点`；终局报告纯文本三段协议同理。
**验证状态**：⭐⭐ 二次验证（同项目两处独立成立；7B 会写 `5|`、照抄示例占位词「标签：」，解析器容错后均无碍版式）
**操作规则**：① 编号+分隔符开段，客户端按编号对表；② 提示词里的示例标签用真实词（手部动作：）不用元词（标签：），元词会被小模型字面照抄；③ 解析器容忍单数字编号、折行、副标题误装正文。
**反例/边界**：强模型（gpt-5.5）下协议服从近乎完美，此法的价值主要在弱模型兜底。

### 2. 扫码链接是硬预算资源

**核心**：URL 进二维码后每个字符都占物理密度；中文经 deflate 压缩率低，叙事全文是不可压缩的底盘——可选字段必须"按预算上车、超了逐级丢"，且要有物理兜底（码框位按长度自适应放大）。
**来源**：im/ims 字段把长局载荷从 ~1000 推到 1189（v22→v25），176px 框位每模块只剩 2.8px，现场扫不开。
**验证状态**：⚠️ 首次发现（降配 6 档 + 框位 176/184/208 自适应已实测收口，但"可扫密度阈值 v23@176px"沿用主导早期实测，未重新扫码验证）
**操作规则**：① 定硬预算常量并注释依据；② 降配序列按价值排序（叙事>印象>时间线细节）；③ 超预算的最后一档也要有物理兜底。
**反例/边界**：短链服务可根治，但展台离线原则不允许。

### 3. 验证必须用"最长真实载荷"

**核心**：凡有长度/密度上限的功能（二维码、短信、推送、文件名），测试数据必须构造真实上限场景；调试捷径产生的短数据会系统性掩盖超限 bug。
**来源**：QR bug 的直接根因——用调试直跳（空轨迹）的 987 字符载荷验证了"可扫"，真实 8 节点 + AI 全文根本不是这个量级。
**验证状态**：⚠️ 首次教训（本项目内被同一根因打脸一次）
**操作规则**：写验证脚本时先问"这个数据的真实最大形态是什么"，用顶格数据测边界。

### 4. 展台 AI 标配三级降级 + 文案如实分叉

**核心**：云端 API → 本地模型 → 静态模板三级端点链；每级的溯源文案（CLOUD AI / RTX LOCAL AI / 离线档案）和隐私提示必须跟着数据流向走，不许"装本地"或"装读了图"。
**来源**：展馆网络不可信 + 照片隐私是红线；坏 key 实测降级链成立。
**验证状态**：⚠️ 首次实现（降级链坏 key 场景实测通过；展馆真实断网场景待 BW 现场验证）
**操作规则**：① 端点链在配置层表达（fallbackBaseUrl），代码只循环；② 请求参数按模型代际适配（gpt-5 推理系 max_completion_tokens+reasoning_effort，经典模型 max_tokens+temperature），两端互不影响；③ key 只存 gitignore 的 config.local.json，发布脚本强制剔除。

### 5. LM Studio 运维事实清单（非方法论，查错用）

- 守护进程会静默恢复未完成下载任务：同路径手动放模型文件会被索引忽略——删 `.internal/download-jobs-info.json` + 杀进程重启才认
- 浏览器直调必须 `lms server start --cors`，默认不带 CORS 头，fetch 直接 Failed to fetch
- 客户端连续 abort 流式请求会留僵尸生成（首包飙到 20s+），`lms unload --all` 重载即愈
- 模型下载源实测：ModelScope ~11MB/s ≫ HF 直连 2MB/s ≈ hf-mirror（会掉速到 KB 级）
- `lms import -y` 的 `-y` 不跳过移动确认提示，管道喂 Y 也不行，直接把文件放对目录重启守护进程更快
