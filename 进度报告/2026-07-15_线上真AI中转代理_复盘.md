**执行主线**（六步）：

## 方法论沉淀

### 1. OpenAI 区域判定跟调用方 IP 走，CF Worker 不是跳板——钉区域要用 Durable Object

**核心**：Worker 在离调用方最近的 PoP 执行，被封区域发起的调用换不来受支持区域的出口；`locationHint` 的 Durable Object 是 Cloudflare 免费产品线里唯一能钉死执行区域的原语。
**来源**：香港服务器直连 403 → 挂 Worker 仍 403 → 本机（代理出口受支持区域）调同一 Worker 200 → DO(enam) 一次通过。
**验证状态**：⚠️ 首次验证（判据充分：同 Worker 双端对照 + DO 单变量切换）
**操作规则**：① 区域封锁问题先确认"判定跟谁走"（调用方 IP / 密钥 / 账号），用双端对照实验定性；② Smart Placement 优化延迟不管区域，别指望；③ DO 写法：默认导出做门禁，`env.HOP.get(idFromName(...), { locationHint: 'enam' })` 转发，流式 body 直接透传。

### 2. 公开站 AI 的标准解：同域中转 + 分层安全闸

**核心**：密钥永不出服务器（env 文件 600），前端产物零密钥；安全闸各司其职——Origin 校验只防"别的网站白嫖挂站"（可伪造），真防滥用靠模型白名单 + completion 上限 + 每 IP 滑动窗口限流，Worker 层再加中继令牌防直刷。
**来源**：本项目 deploy-proxy 全套（129 行零依赖 Node + systemd + nginx snippet + 一键脚本），闸门逐个实测。
**验证状态**：⚠️ 首次实现（各闸实测拦截生效；真实人流成本消耗待展期观察）
**操作规则**：① SSE 流式必须 `proxy_buffering off`，否则打字机变一次性倾倒；② 客户端断连要联动 abort 上游（`res.on('close', () => ctrl.abort())`），不空烧 token；③ 隐私对齐：照片过境不落盘、日志不记请求体，跟站内文案一字不差。

### 3. 兜底文案的「诚实标注」是免费的诊断探针

**核心**：降级路径的输出带上可识别标记（「镜面朦胧·以直觉代读」vs「读心完成」），线上问题从"感觉不对"变成 30 秒定性——这是昨天为诚实而做的设计，今天以诊断价值二次回本。
**来源**：用户报"片汤话"，凭标记 + 模板逐字比对即锁定兜底路径，无需看一行网络日志。
**验证状态**：⭐⭐ 二次验证（昨日设计动机是诚实，今日独立验证了诊断价值）
**操作规则**：任何多级降级系统，每级输出都应留肉眼可辨的溯源标记（角标/脚标/水印均可），且标记必须跟着真实数据流向走。

### 4. nginx 运维事实（查错用）

- `sites-enabled/` 是通配 include 的活目录——备份文件（`.bak.*`）放里面会被当配置加载，`nginx -t` 报 duplicate listen；备份挪 `/root/nginx-backups/`
- 给流式接口挂反代：`proxy_buffering off` + `proxy_read_timeout` 放宽 + `client_max_body_size` 按最大请求体（照片 dataURL）放宽，三件套一次配齐
